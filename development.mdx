## Fluxo recomendado

<Steps>
  <Step title="Clone o repositório">

```bash
git clone https://github.com/fabricioctelles/skills.git
cd skills
```

  </Step>
  <Step title="Edite ou crie uma skill">

Cada skill fica em `skills/<nome-da-skill>/SKILL.md`.

Inclua:
<ul>
  <li>frontmatter com `name` e `description`</li>
  <li>instruções de workflow</li>
  <li>exemplos de entrada e saída</li>
</ul>

  </Step>
  <Step title="Teste com seu agente">

Execute um prompt real e confirme:
<ul>
  <li>se o agente acionou a skill correta</li>
  <li>se o output está consistente com o objetivo</li>
</ul>

  </Step>
</Steps>

## Documentação com Mintlify

Para pré-visualizar esta documentação localmente:

```bash
npm i -g mint
mint dev
```

Por padrão, o preview abre em `http://localhost:3000`.

## Validações úteis

```bash
mint broken-links
mint validate
```

## Boas práticas para contribuição

- Mantenha cada skill focada em um problema específico.
- Use linguagem objetiva e comandos reproduzíveis.
- Atualize exemplos quando APIs ou ferramentas mudarem.
- Evite duplicar conteúdo entre skills; prefira referências claras.

<Info>
  Antes de publicar mudanças amplas, revise também o `AGENTS.md` para manter alinhamento com o contexto e as regras do projeto.
</Info>
